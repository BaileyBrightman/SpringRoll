springroll.Game Class

Extends springroll.Application
Defined in: springroll.Game:12
Module: Game

A game extends the main application and provides some game specific convenience function and additional events. Most importantly it provides preload functionalty though the state manager. Assume loading a single configuration JSON file.

Constructor

springroll.Game

(
  • [options]
)

Defined in springroll.Game:12

Parameters:

  • [options] Object optional

    The collection of options, see Application for more options.

    • [name] String optional

      The name of the game

    • [configPath='assets/config/config.json'] String optional

      The path to the default config to load

    • [forceMobile=false] Boolean optional

      Manually override the check for isMobile (unminifed library version only)

    • [updateTween=true] Boolean optional

      Have the application take care of the Tween updates

    • [fps=60] Int optional

      The framerate to use for rendering the stage

    • [raf=true] Boolean optional

      Use request animation frame

    • [versionsFile] String optional

      Path to a text file which contains explicit version numbers for each asset. This is useful for controlling the live browser cache. For instance, this text file would have an asset on each line followed by a number: assets/config/config.json 2 this would load assets/config/config.json?v=2

    • [cacheBust=false] Boolean optional

      Override the end-user browser cache by adding "?v=" to the end of each file path requested. Use for developmently, debugging only!

    • [basePath] String optional

      The optional file path to prefix to any relative file requests this is a great way to load all load requests with a CDN path.

    • [resizeElement] String | DOMElement | Window optional

      The element to resize the canvas to

    • [uniformResize=true] Boolean optional

      Whether to resize the displays to the original aspect ratio

    • [maxAspectRatio] Number optional

      If doing uniform resizing, optional parameter to add a maximum aspect ratio. This allows for "title-safe" responsiveness. Must be greater than the original aspect ratio of the canvas.

    • [queryStringParameters=false] Boolean optional

      Parse the query string paramenters as options

    • [debug=false] Boolean optional

      Enable the Debug class

    • [minLogLevel=0] Int optional

      The minimum log level to show debug messages for from 0 (general) to 4 (error), the Debug class must be used for this feature.

    • [debugRemote] String optional

      The host computer for remote debugging, the debug module must be included to use this feature. Can be an IP address or host name.

    • [updateTween=false] Boolean optional

      If using TweenJS, the Application will update the Tween itself

    • [canvasId] String optional

      The default display DOM ID name

    • [display] Function optional

      The name of the class to instaniate as the display (e.g. springroll.PixiDisplay)

    • [displayOptions] Object optional

      Display-specific options

    • [crossOrigin=false] Boolean optional

      Used by springroll.PixiTask, default behavior is to load assets from the same domain.

    var game = new springroll.Game();
    game.on('loaded', function(){
        // Ready to use!
    });
